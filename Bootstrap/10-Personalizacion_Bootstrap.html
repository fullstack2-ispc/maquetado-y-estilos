Tema 10

Personalización de Bootstrap

Tres enfoques, desde el más simple hasta el profesional completo.

Los tres métodos

¿Cómo se personaliza Bootstrap?

Hay tres formas, de menor a mayor potencia. Para este tema trabajamos con los dos primeros; el tercero lo veremos cuando usemos bundlers.

Método 1

CSS Custom Properties

Sobreescribir las variables --bs-* en :root o en el selector del componente.

  • ✅ Sin instalación ni build step
  • ✅ Funciona con CDN
  • ✅ Ideal para proyectos simples
  • ⚠️ El CSS de Bootstrap se descarga completo
  • ⚠️ No se pueden eliminar componentes no usados
:root { --bs-primary: ... }
Método 2

Sass con variables

Sobreescribir las variables $primary, $font-family-base antes de importar Bootstrap.

  • ✅ El CSS generado es más pequeño
  • ✅ Permite desactivar módulos enteros
  • ✅ Estándar profesional
  • ⚠️ Requiere Node.js y un bundler
  • ⚠️ Curva de aprendizaje inicial
$primary: #1a1a2e;
Método 3

Bootstrap Customize Tool

Interfaz web oficial que genera un CSS de Bootstrap ya configurado para descargar.

  • ✅ Sin código ni instalación
  • ✅ Útil para prototipos rápidos
  • ⚠️ No es reproducible en equipo
  • ⚠️ Difícil de mantener a largo plazo
getbootstrap.com → Customize
¿Cuál usar en producción? En proyectos reales con equipo y control de versiones, el estándar es el Método 2 (Sass). El Método 1 es perfecto para aprender, para proyectos pequeños, o cuando no podés configurar un bundler. En Angular, el archivo src/styles.scss actúa como el punto de entrada donde aplicás cualquiera de los dos métodos.

La mecánica

La cascada CSS y el orden de los archivos

Entender por qué funciona es tan importante como saber cómo hacerlo.

El orden en el <head>

<!-- 1. Bootstrap PRIMERO -->
<link href="bootstrap.min.css" rel="stylesheet">

<!-- 2. Nuestro CSS DESPUÉS -->
<link href="10-Personalizacion_Bootstrap.css" rel="stylesheet">

CSS Custom Properties respetan la cascada normal. El último valor declarado en el mismo selector gana. Nuestro :root en archivo-principal.css viene después del de Bootstrap, así que nuestras variables sobreescriben las originales.

Dos niveles de sobreescritura

Nivel global → :root

Afecta todos los componentes

:root {
  --bs-primary: #1a1a2e;
}

Nivel componente → selector específico

Afecta solo ese componente

.card {
  --bs-card-border-radius: 8px;
}
.navbar {
  --bs-navbar-padding-y: 1rem;
}
Regla de oro Si el cambio es de identidad visual (color de marca, tipografía, radios globales): usá :root. Si es un ajuste puntual de un componente específico: usá el selector del componente. Mezclar ambos sin criterio genera CSS difícil de mantener.

Tamaño fluido con clamp()

Colección Invierno 2026

font-size: clamp(2rem, 5vw, 4rem);
/*
  clamp(mínimo, preferido, máximo)
  · móvil pequeño   → 2rem  (mínimo garantizado)
  · viewport medio  → 5vw   (escala suave con el viewport)
  · pantalla grande → 4rem  (máximo permitido)
  Reemplaza 3 media queries con una sola línea de CSS.
*/
Buena práctica: preconnect + display=swap Siempre cargá Google Fonts con rel="preconnect" en los dos dominios (googleapis.com y gstatic.com). Agregá display=swap en la URL de la fuente. Resultado: el navegador prepara la conexión antes de necesitarla, y el texto se muestra con la fuente del sistema mientras carga, eliminando el "flash de texto invisible".

Imágenes de catálogo

aspect-ratio + object-fit: la solución a la grilla rota

Sin aspect-ratio, cada imagen con distinto tamaño quiebra la grilla. Con dos líneas de CSS, todas tienen la misma altura visual.

aspect-ratio: 3/4

object-fit: cover

object-position: top center

Todas iguales ✔

.img-producto {
  width: 100%;
  aspect-ratio: 3 / 4;          /* proporción retrato estándar en moda  */
  object-fit: cover;             /* recorta sin deformar                 */
  object-position: top center;  /* prioriza la parte superior (la cara) */
  display: block;                /* elimina el espacio inferior de img   */
}

Para cuando uses bundlers

El método profesional: Sass

Cuando el proyecto usa Vite, Webpack o Angular, el estándar es sobreescribir las variables Sass de Bootstrap antes de importarlo.

Archivo: src/styles.scss (Angular) o src/main.scss (Vite) 

// ❶ Variables Sass de Bootstrap — ANTES de importar Bootstrap
$primary:                #1a1a2e;
$danger:                 #e94560;
$font-family-sans-serif: 'Inter', system-ui, sans-serif;
$border-radius:          4px;
$border-radius-lg:       8px;
$enable-shadows:         false;
$enable-negative-margins: true;

// ❷ Funciones de Bootstrap (requerido antes de variables-map)
@import 'bootstrap/scss/functions';

// ❸ Variables de Bootstrap
@import 'bootstrap/scss/variables';
@import 'bootstrap/scss/variables-map';

// ❹ Opciones y mixins
@import 'bootstrap/scss/mixins';

// ❺ Solo los módulos que usamos (en vez de @import 'bootstrap')
@import 'bootstrap/scss/root';
@import 'bootstrap/scss/reboot';
@import 'bootstrap/scss/grid';
@import 'bootstrap/scss/navbar';
@import 'bootstrap/scss/card';
@import 'bootstrap/scss/buttons';
@import 'bootstrap/scss/forms';
@import 'bootstrap/scss/badge';
@import 'bootstrap/scss/utilities/api'; // siempre al final

// ❻ Nuestros estilos al final
@import 'styleshop/variables';
@import 'styleshop/componentes';
¿Por qué Sass genera un CSS más pequeño? Con CSS Custom Properties, el navegador descarga el CSS completo de Bootstrap y aplica las variables en tiempo de ejecución. Con Sass, los valores se propagan en tiempo de compilación: si no importás accordion.scss, no aparece en el CSS final. Un proyecto real puede reducir el peso de Bootstrap al 30–40% del archivo completo.

Checklist

Estándar profesional de personalización

✅ Siempre hacer

  • Cargar Bootstrap antes de tu CSS propio
  • Definir las variables de marca con prefijo propio (--ss-*)
  • Sobreescribir siempre el par --bs-primary + --bs-primary-rgb
  • Sobreescribir en :root para cambios globales
  • Sobreescribir en el selector para cambios de componente
  • Personalizar el color de foco (accesibilidad)
  • Usar preconnect + display=swap en Google Fonts
  • Definir aspect-ratio en imágenes de catálogo

❌ Nunca hacer

  • Editar los archivos de Bootstrap directamente
  • Cargar tu CSS antes de Bootstrap
  • Usar !important en estilos de componentes
  • Mezclar colores hardcodeados con variables CSS
  • Eliminar el outline de foco (outline: none)
  • Duplicar estilos de Bootstrap manualmente
  • Usar transition: all en componentes interactivos
  • Cargar Google Fonts sin display=swap
